# Rust固有の観点

以下のRust固有の観点も追加で確認してください：

## 所有権・借用
- 不要なclone()の使用
- ライフタイムの適切さ
- 借用チェッカーを回避するための不適切なパターン

## エラー処理
- unwrap()/expect()の不適切な使用（ライブラリコード）
- エラー型の適切な設計
- `?` 演算子の活用

## イディオマティックRust
- イテレータの活用（forループよりもmap/filter/collect）
- パターンマッチの活用
- Option/Resultの適切な処理
- derive マクロの活用

## パフォーマンス
- 不要なアロケーション（String vs &str）
- Vecの事前容量確保（with_capacity）
- Cowパターンの検討

## 安全性
- unsafe コードの正当性
- データ競合の可能性
- パニックの可能性がある操作
